# Security Policy

## Reporting a Vulnerability

The HanDl-ng organization takes security seriously. We appreciate your efforts to responsibly disclose vulnerabilities and will make every effort to acknowledge your contributions.

**Please do not report security vulnerabilities through public GitHub issues or discussions.**

Instead, please report security vulnerabilities using one of the following methods:

- **GitHub Security Advisories**: Open a [private security advisory](https://github.com/HanDl-ng/Discussions/security/advisories/new) in this repository.
- **Direct contact**: Reach out to the organization maintainers directly via GitHub.

### What to Include in Your Report

Please include the following information to help us understand and resolve the issue quickly:

- A description of the vulnerability and its potential impact
- The affected project(s) and version(s)
- Step-by-step instructions to reproduce the issue
- Any proof-of-concept code, if applicable
- Any suggested mitigations or fixes

## Response Process

1. We will acknowledge receipt of your report within **72 hours**.
2. We will investigate the issue and keep you informed of our progress.
3. Once a fix is available, we will coordinate a disclosure timeline with you.
4. We will publicly acknowledge your responsible disclosure (unless you prefer to remain anonymous).

## Scope

This security policy applies to all repositories and projects under the [HanDl-ng organization](https://github.com/HanDl-ng).

## Out of Scope

The following are considered out of scope:

- Issues in third-party dependencies (please report those to the respective maintainers)
- Social engineering attacks
- Physical security issues

## Thank You

We value the security community's contributions in keeping our projects safe. Thank you for helping us maintain the security of HanDl-ng projects.
